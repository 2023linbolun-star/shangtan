# Product Spec — 商探AI (ShangTan AI)

> **版本：** v2.0
> **创建日期：** 2026-03-17
> **更新日期：** 2026-03-24
> **Slogan：** AI 替你做电商，你只管收钱

---

## 产品概述

商探AI 是一个**全自动电商操盘系统**（电商自动驾驶）。系统自主扫描全网趋势发现商品机会，自动评估市场可行性并做出"做/不做"决策，自动在1688匹配供应商建立一件代发链路，自动生成各平台内容（短视频脚本/种草笔记/商品Listing），自动发布到抖音/小红书/公众号等渠道获取免费流量，自动处理客服咨询，并基于真实销售数据自动优化——赚钱的品加量，不赚钱的品淘汰。

**目标用户**：所有人。完全不懂电商的新手（系统从零开始替你做）、有经验但想拓展新平台的卖家（做淘宝的想做抖音）、想尝试新品类的卖家（卖瑜伽裤的想试电脑包）——在不熟悉的领域，所有人都是小白。系统不区分用户经验，统一全自动运行，根据用户起点自动规划路径。

**核心价值**：用户只需提供店铺账号，系统替代传统5人电商团队（选品+内容+运营+客服+数据分析）全自动运转。零库存风险（一件代发），零广告预算起步（内容驱动免费流量），每个环节支持"自动/审核"切换，用户可随时接管任意模块。

**支持平台**：
- Phase 1（v2.0上线）：抖音（有视频发布API，主力内容+销售）、小红书（种草，一键复制发布）、1688（供应链API）
- Phase 2：淘宝（搜索承接）、拼多多（价格敏感）、公众号（私域沉淀，有发布API）
- Phase 3：视频号、快手、京东

## 应用场景

- **纯小白零基础起步**：大学生小王想做电商但完全不懂。注册商探AI，选择"完全新手"，点击"启动AI"。系统自动扫描到"冰丝防晒衣"趋势上升，评估利润率47%、竞争度低，自动在1688找到¥15.8/件的代发供应商，生成3条抖音视频脚本和2篇小红书笔记，自动发布到抖音（API），小红书给小王推送一键复制通知。3天后第一单成交，系统自动将订单转发给供应商发货。小王全程没做任何决策。

- **淘宝老手拓抖音**：做了3年淘宝的老李想做抖音但不懂短视频。他把淘宝店铺接入商探AI，系统分析他现有的瑜伽裤商品，自动生成适合抖音的30秒带货脚本（Hook-痛点-产品-信任-CTA结构），自动通过抖音API发布视频。老李觉得自己写文案更好，把"内容生成"切成审核模式，其他环节保持全自动。

- **成功品类快速复制**：卖家小张在抖音卖瑜伽裤月销5万。系统分析他的成功模式，自动推荐"运动内衣"作为拓品方向（同目标人群、类似供应链、利润率更高），自动走完选品→供应商→内容→上架全流程，小张只在"新品上架"环节点了一下确认。

- **批量内容矩阵**：代运营团队管理10家店铺。每家店铺的选品、内容、客服全部由商探AI自动化。团队只需在驾驶舱看各店铺的营收数据，对表现差的品点"淘汰"，对表现好的品点"加量"。1个人管10家店。

- **全自动被动收入**：白领小陈白天上班，晚上想搞副业。他花5分钟注册+引导，之后什么都不管。系统每天自动选品、发内容、处理客服、发货。小陈每周日晚上看一下周报，发现有3个品在赚钱、2个被自动淘汰了。月底结算净收入¥3,200。

## 功能需求

### 核心功能

**M0 — 自动驾驶控制中心（Autopilot Cockpit）**

- 自动化闭环可视化：7个模块节点（发现→评估→供应链→内容→发布→客服→分析）排列成椭圆轨道，数据流动时有动画连线
- 全局自动/审核切换：一键切换全局模式，或按模块独立切换
- 关键指标实时展示：今日营收/利润/在管商品数/已发内容数/订单数/转化率，数字变化有动画
- 实时活动流：AI 每一个动作实时显示（"发现上升品类: 冰丝防晒衣 +340%"、"已生成3条抖音脚本"、"自动回复4条咨询"），等宽时间戳+彩色模块标签
- 待审核队列：需要用户确认的项目集中展示，按紧急程度排序，支持预览/通过/编辑/拒绝
- 紧急停止：一键暂停所有自动化操作

**M1 — 趋势发现引擎（Trend Discovery）**

- 自动扫描：每4小时扫描抖音热搜/百度指数/微信指数/淘宝搜索趋势/季节性日历 → 评分排序 → 输出趋势机会列表
- 数据源：HotList API（免费，覆盖抖音/微博/B站）、ALAPI热榜（免费200次/天）、pytrends、淘宝联盟搜索API、季节性日历JSON（618/双11/年货节等硬编码）
- 趋势卡片：关键词+迷你趋势图+机会评分+平台标签
- 自动模式：每日自动将Top 3趋势送入评估流水线
- 审核模式：展示趋势列表，用户挑选哪些进入评估

**M2 — 市场评估 + GO/NO-GO 决策引擎（Market Evaluation）**

- 多平台数据采集：淘宝联盟API + 多多进宝API + 抖音精选联盟API（已有ScoutAggregator）+ 新增1688搜索API（估算采购成本）
- 确定性评分（AI调用前）：五维评分模型（需求热度0.25 + 利润空间0.25 + 竞争可行性0.20 + 趋势动量0.15 + 供应链匹配0.15）
- 硬过滤淘汰：净利率<15% / 需特殊资质 / 月搜索量<100 / Top3垄断>80% / 高侵权风险 → 直接NO-GO
- AI深度分析：仅对通过确定性评分（>0.4）的商品调用AI分析，节省成本
- GO/NO-GO输出：评估报告卡片 + GO/NO-GO/有条件 徽章 + 置信度百分比 + 五维雷达图
- 自动模式：GO自动推进，NO-GO自动跳过
- 审核模式：所有决策暂停等确认

**M3 — 供应商匹配引擎（Supplier Sourcing）**

- 1688自动搜索：根据选品关键词在1688搜索，筛选支持一件代发的供应商（牛皮≥2星、回复率>80%、支持单件发货）
- 排名Top 3：按价格/发货速度/退货政策/交易量加权排序
- 成本拆解：采购价 + 国内快递(3-8元) + 包装(1-3元) + 平台佣金 = 利润率
- API：1688开放平台 `alibaba.open/search` + `product/detail`（个人开发者实名注册可用）
- 自动模式：选择满足所有质量阈值的最便宜供应商
- 审核模式：用户从Top 3中选择

**M4 — 内容工厂（Content Factory）**

- 多平台自动生成：
  - **抖音**：30-45秒视频脚本，Hook(3秒抓眼球)-痛点-产品展示-信任背书-CTA结构，3个变体A/B测试，完播率>一切
  - **小红书**：70/20/10内容配比（70%价值/20%互动/10%推广），颜值优先，800-1500字+emoji标题+标签推荐
  - **淘宝**：关键词密集标题(60字内) + 五点式卖点 + 详情页文案
  - **拼多多**：短平快listing，价格优先，信任词（包邮/正品/厂家直销）
  - **公众号**：长文章用于私域沉淀，70%+价值内容
- 违规检测：广告法禁用词 + 平台违禁词 + 虚假宣传 + 侵权风险 → 绿/黄/红三级
- 商品图片生成：AI抠图+换背景+添加文案（可灵AI）
- 自动模式：生成后直接进入发布队列
- 审核模式：每条内容暂停等编辑/确认（新用户默认）

**M5 — 发布引擎（Publishing Engine）**

- 多平台发布：
  - **抖音视频**：通过官方视频发布API全自动（`/video/upload/` + `/video/create/`，个人开发者实名认证可申请）
  - **小红书**：无发布API → 生成"一键复制包"（标题+正文+标签+配图建议），推送通知用户粘贴发布
  - **公众号文章**：通过官方API全自动（`/cgi-bin/draft/add` + `/freepublish/submit`）
  - **视频号**：无API → 同小红书一键复制包
  - **淘宝上架**：生成CSV数据包（淘宝格式），用户上传卖家中心（后续接ERP工具自动化）
  - **拼多多上架**：商家API上架（需商家账号）或CSV导出
  - **抖店上架**：企业API（后续）或生成数据包手动上传
- 发布日历：按各平台最佳发布时间自动排期
- 自动模式：API平台自动发布，手动平台推送通知
- 审核模式：所有平台暂停等确认

**M6 — 订单履约监控（Order & Fulfillment）**

- 订单监控：追踪各平台订单状态
- 一件代发转发：订单到达 → 自动生成转发指令给1688供应商（初期手动操作，后续1688 API自动下单）
- 物流追踪：快递100/快递鸟 API 追踪发货状态
- 异常检测：发货延迟>48h、特定商品高退货率 → 自动告警

**M7 — AI客服（CS Brain）**

- 平台接入：淘宝千牛（API最开放，可自动回复）、其他平台生成建议回复
- 三级置信度：>0.9 自动发送、0.7-0.9 建议确认、<0.7 转人工
- 知识库：从listing自动提取商品信息 + 退换货政策 + FAQ，pgvector语义搜索
- 投诉升级：涉及法律/健康/欺诈的投诉永不自动回复，必须转人工

**M8 — 数据分析 + 自我进化引擎（Performance & Learning）**

- 单品经济模型：营收 - 采购成本 - 运费 - 平台佣金 - 退货损失 = 净利润
- 加量/维持/淘汰 决策（上架7天后）：
  - 淘汰：0订单 或 退货率>40% 或 净利润为负
  - 维持：1-10订单，正利润
  - 加量：>10订单，净利润>10元/件，退货率<20%
- 广告触发条件：仅在自然验证通过后触发（≥5个自然订单 + 净利率≥25% + 退货率<30%），支持千川/直通车/多多搜索 API
- 反馈闭环：每个AI决策记录结果，好的决策加入few-shot正例，差的决策提取失败模式
- AI日报/周报：自动推送到Telegram/钉钉

### 辅助功能

**P0 — 基础平台（Platform）**

- 用户系统：手机号+短信验证码注册，邀请码制（内测）
- 多店铺管理：每用户管理多个店铺，每店铺独立配置运营模式/审批门/风险阈值
- 平台凭证管理：OAuth流程对接各平台，Fernet加密存储token
- 通知系统：站内 + Telegram + 钉钉 + Email
  - 胜利通知（新成交/里程碑）：绿金色toast+微型礼花
  - 警告通知（表现不佳/异常）：琥珀色toast
  - 待操作通知（需审核/需确认）：驾驶舱状态条计数+侧边栏徽章

## UI 布局

### 整体布局

全局暗色主题。左侧固定侧边栏（~240px），右侧内容主区域（自适应）。顶部sticky状态条。

**视觉方向**：AI驾驶舱风格，不是管理后台。深色底色（近黑蓝 `oklch(0.13 0.02 260)`），电光青色为活跃进程，绿色为收入/成功，琥珀色为待审核/警告。活跃元素带发光 `box-shadow`，数字等宽+tabular-nums，毛玻璃顶栏/侧边栏，边框极淡。参考 Tesla FSD + Linear.app + Bloomberg。

### 左侧 — 侧边栏

- 顶部：商探AI Logo + "Autopilot"标识
- 导航菜单：
  - 驾驶舱（首页）
  - 趋势发现
  - 在管商品
  - 内容工厂
  - 发布中心
  - 订单履约
  - AI客服
  - 数据分析
  - 设置（店铺/凭证/自动化/风险）
- 每个导航项旁显示模块状态指示点（绿=自动运行，琥珀=有待审核项，灰=空闲）
- 底部：折叠/展开

### 右侧 — 内容主区域

**驾驶舱页面（默认首页，核心页面）：**

- Section A — 状态条（sticky，48px）：AI状态、运行进程数、今日营收、待审核数、上次AI动作时间
- Section B — 自动化闭环可视化（400px）：7个模块节点椭圆轨道，SVG动画连线，数据流动发光点，中心显示在管商品数
- Section C — 关键指标条（80px）：6个指标卡片横排（营收/利润/商品/内容/订单/转化率），迷你趋势图+CountUp动画
- Section D — 模块状态网格（2×4）：每模块一张卡片，含状态指示+自动/审核分段切换器+一行状态文字。首位"全局自动驾驶"大卡片
- Section E — 实时活动流（~300px）：AI动作实时日志，等宽时间戳+彩色模块标签，新条目顶部滑入
- Section F — 待审核队列：需确认项目卡片列表，支持预览/通过/编辑/拒绝

**模块详情页（统一模式）：**

- 自动模式视图：监控面板式（状态卡+"AI正在运行此模块" + 最近动作日志 + 模块指标 + "手动触发"按钮）
- 审核模式视图：交互面板式（审核队列 + 标准操作UI + 审核/通过/拒绝）
- 模式切换：审核→自动需确认弹窗；自动→审核即时生效

### 控件规范

- 所有界面暗色底，卡片用略浅暗色（`oklch(0.17 0.02 260)`）
- 活跃/运行中元素带cyan发光（`box-shadow: 0 0 15px oklch(0.75 0.15 195 / 0.3)`）
- 自动/审核切换器：分段控制（[自动|审核]），自动=青色光晕，审核=琥珀光晕
- 加载状态：shimmer扫光动画（不是转圈）
- 过程展示：分步进度列表，每步有状态图标（✓完成 / ⏳执行中 / ○等待），步骤间有动画连接线
- 图表：Recharts，暗色适配
- 动效库：framer-motion

## 用户使用流程

### 新用户引导流程（Onboarding）

1. 注册（手机号+短信）
2. 欢迎页（全屏暗色+粒子网络动画）
3. 平台选择（平台logo卡片，选中发光边框）
4. 预算+风险偏好（保守/平衡/激进）
5. 品类偏好（可跳过，让AI自己发现）
6. 自动化级别（全自动🤖 / 有引导🤝 / 手动+AI辅助🛠️）
7. 启动时刻：全屏大按钮"LAUNCH AI ▶"脉冲光晕动画，点击后逐行显示系统初始化：
   - "✓ 连接平台API..."
   - "✓ 加载市场数据模型..."
   - "✓ 初始化发现引擎..."
   - "✓ 内容生成就绪..."
   - "✓ 客服AI上线..."
   - "✓ 所有系统就绪。"
   过渡到驾驶舱。

### 日常自动运行流程

1. 系统每4小时自动扫描趋势
2. 发现机会 → 自动评估 → GO的品自动进入供应商匹配+内容生成+发布
3. 用户收到每日摘要推送（Telegram/钉钉）：新机会X个、发布X条、待审核X项
4. 需要审核的项目在驾驶舱待审核队列，用户一键通过或编辑
5. 订单自动转发供应商发货
6. 每周业绩汇总 + 加量/淘汰建议
7. 随时间推移，连续5次不修改通过的步骤，系统建议切换为自动

### 用户接管某个环节

1. 在驾驶舱或模块详情页，将目标模块切换为"审核"模式
2. 该模块所有AI输出暂停等用户确认
3. 用户可预览、编辑、通过或拒绝
4. 其他模块继续全自动运行
5. 随时可切回自动

## 技术说明

- **前端**：Next.js 14 + React 18 + TypeScript + Tailwind CSS + Shadcn UI（全局暗色主题重写）+ framer-motion（动效），状态管理 Zustand，图表 Recharts
- **后端**：FastAPI (Python 3.11) + SQLAlchemy 2.0 + Alembic，异步优先
- **数据库**：PostgreSQL 16 + pgvector（向量搜索），Redis 7（Pub/Sub + Celery Broker + 缓存 + 限速）
- **任务队列**：Celery 5 + Redis + Celery Beat（定时调度），3个队列（agents / platform_io / fast）
- **实时事件**：Redis Pub/Sub → SSE推送到前端（替换内存EventBus+轮询）
- **文件存储**：阿里云 OSS
- **AI 服务**：Claude API（Sonnet/Haiku）+ DeepSeek API + 可灵AI，统一通过 ai_engine.py 封装
- **平台集成**：三层适配器模式 — ReadAdapter（数据读取）+ WriteAdapter（API发布/上架）+ ManualAdapter（生成手动操作指令）
- **凭证管理**：Fernet对称加密存储每店铺平台OAuth凭证，密钥仅在环境变量
- **部署**：阿里云新加坡 ECS + systemd + Nginx
- **安全**：JWT认证、多租户数据隔离（所有查询必须带store_id/user_id）、速率限制、XSS/SQL注入防护

### 自动/审核信任积累机制

| 阶段 | 交互次数 | 默认行为 |
|------|---------|---------|
| 新手（0-10次） | 所有关键步骤=审核 | 仅趋势扫描和数据采集自动 |
| 学习（10-50次） | 连续5次无修改通过的步骤提示切自动 | 内容+供应商可切自动 |
| 信任（50+次） | 用户可启用全自动 | 仅加量/淘汰决策默认审核 |
| 随时可切 | 永远 | 用户可将任意步骤切回审核 |

### 设计原则

1. **系统决策，系统执行** — 不是用户告诉系统做什么，而是系统自己决定做什么
2. **免费流量优先** — 内容驱动自然流量，广告是验证后的加速器不是起步方式
3. **零风险启动** — 一件代发不压货，不投广告不烧钱，试错成本约等于零
4. **过程可见** — 用户要看到AI在替他工作，不是黑箱出结果
5. **收权无门槛，放权要确认** — 切回审核即时生效，切到自动需确认
6. **数据驱动进化** — 每个决策记录结果，系统越用越准
7. **降级容错** — 任何外部API失败不阻塞流程，有API用API，没API给一键复制
