# Product Spec — 商探AI (ShangTan AI)

> **版本：** v1.0
> **创建日期：** 2026-03-17
> **Slogan：** AI 洞察商机，全链路掌控

---

## 产品概述

商探AI 是面向国内电商卖家的 **AI 全链路运营一体化工具**，覆盖从选品调研、内容生成、商品上架、广告投放、日常运营、智能客服到数据分析的完整经营闭环。通过 AI 自动化替代传统电商团队中选品部、设计部、运营部、客服部的重复性工作。

**目标用户**：国内电商卖家，包括个人新手卖家（1人运营、预算有限、需要选品指导和降低入行门槛）、中小卖家（1-5人团队、需要提效降本和多平台管理）、电商团队（5-20人、需要数据驱动决策和规模化运营）、工厂转电商（有供应链但不懂线上运营）、以及代运营公司（管理多品牌/店铺、需要批量操作和标准化流程）。

**核心价值**：一个人 + 商探AI = 一个电商运营团队。用户只需输入商品信息和经营条件，AI 就能完成市场分析、内容创作、Listing 优化、广告方案、客服回复、经营报告等全链路工作，大幅降低电商运营的人力成本和专业门槛。

**支持平台**：抖音电商（P0）、淘宝/天猫（P0）、拼多多（P1）、小红书（P1）、京东（P2）、快手（P2）。v1.0 以手动输入 + AI 分析为主，后续逐步对接各平台开放 API。

## 应用场景

- **新手选品**：刚入行的个人卖家小王想在抖音卖货，但完全不知道选什么品。他在商探AI 输入"瑜伽裤"，AI 分析市场规模、竞争格局、进入难度，推荐了 3 个低竞争高利润的细分方向，附带 1688 货源建议和利润测算，小王当天就确定了方向。

- **内容批量生成**：中小卖家小李有 20 款商品需要在小红书种草推广，以前请写手一篇 200 元，现在用商探AI 内容工厂，输入商品信息和目标风格，5 分钟生成 20 篇种草笔记（含标题、正文、标签、配图建议），质量不输人工，成本几乎为零。

- **多平台上架**：电商团队运营小张管理 3 个平台的店铺，同一款商品在淘宝、抖音、拼多多的标题规则和品类映射都不同。使用商探AI 上架中心，录入一次商品信息，AI 自动优化各平台标题、关键词、描述，一键导出 3 个平台的上架文件，还自动检测违规风险。

- **竞品实时监控**：卖家老赵发现竞品突然降价 20%，如果不及时跟进就会丢失市场。商探AI 竞品监控功能在竞品价格变动 >10% 时自动推送钉钉告警，并给出 AI 分析（降价原因、建议应对策略），老赵第一时间做出决策。

- **AI 智能客服**：代运营公司管理 10 家店铺，每天上百条客户咨询。商探AI 客服大脑统一接收多平台消息，AI 自动回复常见问题（查物流、退换货政策、尺码咨询），高置信度直接发送，低置信度转人工，客服人力减少 70%。

## 功能需求

### 核心功能

**M1 — 选品雷达（Product Scout）**

- 市场洞察：用户输入关键词/商品链接/品类 → AI 分析市场规模、竞争格局、价格带、进入难度（0-100分）、季节性、细分机会 → 输出结构化分析报告
- 爆款追踪：用户浏览各平台热销商品和新晋爆款 → AI 分析爆款原因（价格/内容/季节/大促）→ 输出跟款可行性评估和建议（建议跟/谨慎跟/不建议跟）
- 竞品监控：用户添加竞品店铺/商品链接 → 系统持续追踪价格/销量/评分/排名变化 → 异常变动自动告警（站内通知 + Telegram + 钉钉）
- 选品推荐引擎：用户设定目标平台、品类偏好、预算范围、利润率要求、经营模式、风险偏好 → AI 综合评分（需求热度×0.25 + 利润空间×0.25 + 竞争可行性×0.20 + 趋势潜力×0.15 + 供应链匹配度×0.15）→ 输出推荐列表 + 五维雷达图 + 差异化建议 + 1688货源建议 + 风险提示
- 利润计算器：用户输入采购成本/物流/包装/售价/平台佣金/广告费/退货率 → 系统实时计算毛利率、净利率、盈亏平衡点、ROI → 输出不同售价的利润对比表（内置各平台各品类佣金表）
- 趋势雷达：系统监测百度指数/微信指数/抖音搜索趋势/社交媒体热点 → AI 分析趋势上升品类/关键词 → 输出热点预警和 AI 趋势周报

**M2 — 内容工厂（Content Factory）**

- 短视频脚本生成：用户选择商品 + 视频类型（商品展示/开箱种草/使用教程/对比评测/故事剧情/口播带货）+ 目标平台 + 风格调性 → AI 生成完整分镜脚本 + 画面描述 + 口播文案 + BGM 建议 + 封面标题 + 发布建议
- 种草笔记生成：用户输入商品信息 + 选择笔记类型（好物分享/合集推荐/测评对比/教程攻略/场景种草）→ AI 生成标题（多版本含 emoji）+ 正文 + 配图建议 + 标签推荐 + SEO 关键词
- 商品图片生成：用户上传商品原图 + 选择图片类型（白底主图/场景图/卖点图/对比图/尺寸图）+ 目标平台 → AI 自动抠图 + 换背景 + 添加文案 + 排版 → 生成多版本供选择
- 直播话术生成：用户输入商品信息 + 直播时长 + 商品数量 + 主播风格 → AI 生成完整直播脚本（开场/讲解/逼单/互动/过渡/下播话术，带时间轴标注）
- 广告素材生成：用户选择目标平台（千川/直通车/聚光/磁力引擎）→ AI 生成广告文案（多组 A/B 变体）+ 广告配图 + 视频脚本
- 素材库：所有生成素材自动按商品/平台/类型/日期归档，支持收藏、标签、搜索、批量导出、素材效果追踪

**M3 — 上架中心（Listing Hub）**

- 商品信息管理：统一管理 SKU、标题、品牌、品类、规格参数、成本信息、图片素材、卖点、供应商信息
- Listing 优化器：用户选择目标平台 → AI 优化标题（按各平台字符限制和关键词规则）、五点式卖点描述、详情页文案、搜索关键词 → 输出优化评分（0-100）+ 优化前后对比
- 多平台格式导出：一份商品信息自动适配淘宝数据包 CSV / 拼多多 Excel / 抖店 Excel / 京东 Excel / 通用 JSON/CSV，自动裁剪标题、映射品类、适配图片尺寸、检查必填字段
- 关键词工具：用户输入核心词 → AI 拓展长尾词/相关词/下拉词/热搜词 → AI 自动按搜索意图分组（品牌词/品类词/属性词/场景词/长尾词）→ 保存到关键词库
- 违规检测：上架前 AI 扫描内容 → 检测广告法违禁词/平台禁用词/虚假宣传/侵权风险/资质要求 → 输出风险等级（绿色通过/黄色警告/红色阻断）+ 具体问题定位 + 修改建议

### 辅助功能

**M4 — 广告助手（Ad Assistant）**

- 投放方案生成：用户输入商品信息和预算 → AI 生成完整投放方案（投放策略/关键词定向/出价/日预算/时段/素材搭配/预期效果），支持千川/直通车/多多搜索/聚光/磁力金牛
- 广告数据分析：用户上传各平台广告报表（CSV/Excel）→ 系统按计划/单元/关键词/素材维度分析效果（花费/曝光/CTR/转化/ROAS/CPC），展示趋势图表和异常检测
- AI 优化建议：基于广告数据自动生成优化建议（暂停低效词/调价/预算再分配/素材迭代/人群优化）
- A/B 测试管理：创建素材/文案/出价测试计划 → 追踪效果 → 自动计算统计显著性推荐赢家

**M5 — 运营管家（Operations Manager）**

- 智能定价：基于竞品价格 + 成本 + 目标利润率 → AI 推荐最优定价，支持多平台差异化定价
- 库存预警：手动录入库存数量 → 基于日均销量计算可售天数 → 安全库存提醒、补货建议、大促备货计算
- 促销策划：基于电商日历（618/双11/年货节等）→ AI 自动生成促销方案（折扣力度/优惠券策略/满减方案/赠品搭配）+ 效果预估
- 评论管理：手动录入/粘贴评论 → AI 情感分析（正面/中性/负面）+ 差评归因分类 + AI 生成回复模板 + 差评预警 + 好评卖点提炼

**M6 — 客服大脑（CS Brain）**

- 多平台消息接入：v1.0 支持 Email/Telegram Bot/网页在线客服，统一收件箱按店铺/平台/紧急程度分类
- AI 自动回复：基于商品知识库 + 订单信息 + 店铺政策自动生成回复，高置信度(>0.9)自动发送、中置信度(0.7-0.9)人工确认、低置信度(<0.7)转人工
- 知识库：店铺级（退换货政策、发货时效、FAQ）+ 商品级（从 Listing 自动提取），支持语义搜索（pgvector）
- 工单系统：AI 无法处理的问题自动创建工单 → 按优先级分配（紧急/高/中/低）→ SLA 追踪

**M7 — 数据驾驶舱（Data Cockpit）**

- 经营看板：总销售额/总利润/订单量/客单价/退货率/广告ROI/转化率/内容产出量/选品通过率，支持按店铺/平台/商品/时间维度切换
- AI 经营日报/周报：每日自动生成关键指标变化 + 异常告警 + AI 归因分析 + 行动建议，推送到站内/Telegram/钉钉/Email
- 归因分析：销量变化归因（价格/评分/广告/竞品/季节/大促），各渠道贡献分析
- 趋势预测：基于历史数据预测未来 7/30 天销量（考虑季节性/促销日历/广告计划/库存水平）

**P0 — 基础平台（Platform）**

- 用户系统：手机号+短信验证码 / 邮箱+密码注册，邀请码制（内测），用户角色（superadmin/owner/admin/editor/viewer）
- 店铺管理：每用户管理多个店铺，每店铺关联平台和市场，店铺经营 DNA（经营风格/选品偏好/广告策略/内容风格/客服风格）自动调整 AI 输出
- 积分计费系统（运营力）：注册赠送 3000 运营力，每日签到 +100，邀请好友 +500，购买充值 ¥1=100 运营力；各 AI 操作消耗不同运营力（市场洞察 25/短视频脚本 30/Listing 优化 25/AI 客服回复 2 等）；会员等级（新手/银牌/金牌/钻石）对应 9-7 折优惠
- 团队协作：创建团队 → 邀请成员 → 角色权限管理
- 通知系统：站内通知 + Telegram + 钉钉 + Email，支持自定义通知偏好

## UI 布局

### 整体布局

左右两栏布局。左侧为固定暗色侧边栏（约 240px 宽），右侧为内容主区域（自适应填满剩余空间）。顶部有固定顶栏（用户信息、运营力余额、通知、店铺切换）。内容区顶部有面包屑导航。整体采用暗色主题。

### 左侧 — 侧边栏

- 顶部：商探AI Logo + 产品名称
- 导航菜单（按模块分组）：
  - 工作台（首页）
  - 选品雷达（市场洞察 / 爆款追踪 / 竞品监控 / 选品推荐 / 利润计算器 / 趋势雷达）
  - 内容工厂（新建内容 / 内容列表 / 素材库）
  - 上架中心（商品管理 / 关键词工具 / 多平台导出）
  - 广告助手（广告计划 / 数据分析 / A/B 测试）
  - 运营管家（定价管理 / 库存 / 促销 / 评论）
  - 客服大脑（统一收件箱 / 知识库 / 工单）
  - 数据驾驶舱（经营看板 / AI 报告 / 趋势预测）
  - 积分中心（余额 / 交易记录 / 充值 / 签到）
  - 设置（店铺管理 / 团队 / 通知偏好）
- 底部：折叠/展开按钮、使用说明入口

### 右侧 — 内容主区域

- 顶栏：面包屑导航 / 当前店铺切换下拉框 / 运营力余额显示 / 通知铃铛 / 用户头像下拉菜单
- 工作台首页：
  - 引导流程卡片（选品 → 内容 → 上架 → 推广 → 运营 → 客服 → 数据）
  - 各模块快捷入口卡片
  - 待办事项（竞品告警、库存预警、差评通知等）
  - 运营力余额和今日签到
- 各模块页面遵循统一布局：页面标题 + 操作按钮区 + 主内容区（列表/表单/分析结果）

### 控件规范

- 搜索/输入框：铺满内容区宽度，圆角，暗色背景，placeholder 提示文字
- 主操作按钮：醒目样式（品牌色），靠右对齐
- 数据表格：斑马纹，支持排序和筛选
- 卡片组件：圆角、阴影，用于商品卡片/竞品卡片/内容项目卡片/推荐卡片
- AI 分析结果：使用可展开的区块展示，雷达图/趋势图使用 Recharts
- 弹窗/抽屉：用于详情查看和表单编辑
- 加载状态：AI 生成过程显示进度条 + 步骤提示（SSE 实时推送）

## 用户使用流程

### 新用户上手流程

1. 注册账号（手机号/邮箱 + 邀请码）
2. 创建第一个店铺（选择平台、填写店铺信息、设定经营 DNA）
3. 进入工作台首页，查看引导流程卡片
4. 按照「选品 → 内容 → 上架 → 推广 → 运营」链路逐步使用

### 选品分析流程

1. 进入选品雷达 → 市场洞察
2. 输入关键词（如"瑜伽裤"）或粘贴商品链接
3. 点击「开始分析」→ AI 生成市场分析报告（等待约 10-30 秒）
4. 查看分析结果：市场规模/竞争格局/进入难度/细分机会
5. 感兴趣的方向 → 进入选品推荐获取具体商品建议
6. 使用利润计算器验证利润可行性
7. 确定选品 → 添加竞品到监控列表

### 内容创作流程

1. 进入内容工厂 → 新建内容项目
2. 选择内容类型（短视频脚本/种草笔记/直播话术等）
3. 选择或录入商品信息
4. 选择目标平台和风格调性
5. 点击「AI 生成」→ 等待生成完成
6. 预览内容 → 编辑微调 → 保存到素材库
7. 下载或导出使用

### 商品上架流程

1. 进入上架中心 → 新建商品 → 填写基础信息和成本信息
2. 点击「AI 优化 Listing」→ 选择目标平台
3. AI 生成优化后的标题/卖点/描述/关键词 → 查看优化评分和前后对比
4. 点击「违规检测」→ 查看风险报告 → 修改问题内容
5. 选择目标平台 → 点击「导出」→ 下载平台格式文件
6. 在各平台后台上传文件完成上架

## 技术说明

- **前端**：Next.js 14 + React 18 + TypeScript + Tailwind CSS + Shadcn UI（暗色主题），状态管理 Zustand，图表 Recharts
- **后端**：FastAPI (Python 3.11) + SQLAlchemy 2.0 + Alembic，异步优先
- **数据库**：PostgreSQL 16 + pgvector（向量搜索），Redis 7（缓存/会话/Celery Broker）
- **任务队列**：Celery 5 + Redis，5 个队列（default/fast/generation/heavy/scraper）
- **文件存储**：阿里云 OSS（图片/视频/文档/导出文件）
- **AI 服务**：Claude API（Sonnet/Haiku）+ DeepSeek API + 可灵 AI，所有 AI 调用通过 ai_engine.py 统一封装，支持模型路由和降级容错
- **通知推送**：Telegram Bot + 钉钉 Webhook + Email SMTP，实时推送用 SSE
- **部署**：阿里云新加坡 ECS + systemd，数据库每日备份到 OSS
- **数据存储**：需要用户登录，数据存储在 PostgreSQL，UUID 主键，时间使用 UTC
- **API 响应格式**：统一 `{"success": bool, "data": any, "message": str, "error": str|null}`
- **安全**：JWT 认证、速率限制、XSS/SQL 注入防护、平台 API 凭证加密存储、文件上传限制（图片 10MB/视频 100MB）
- **每次 AI 调用前必须检查用户运营力余额，调用失败时退还已扣运营力**

### v1.0 数据来源策略

**核心原则：选品数据必须真实 — AI 只做分析决策，不编造数据。**

| 数据需求 | 数据来源 | 个人可申请 | 优先级 |
|---------|---------|-----------|--------|
| 淘宝/天猫商品+销量+佣金 | 淘宝联盟 API（阿里妈妈） | 是 | P0 |
| 拼多多商品+销量+佣金 | 多多进宝 API | 是 | P0 |
| 抖音商品+佣金 | 聚推客 API（抖音精选联盟） | 是 | P0 |
| 小红书种草热度 | 小红书开放平台 | 是 | P1 |
| 采购成本/物流/包装 | 用户手动输入 | — | P0 |
| 竞品数据 | 用户手动粘贴链接 + 手动刷新 | — | P1 |
| 广告数据 | 用户上传报表 CSV/Excel | — | P1 |
| 京东商品数据 | 京东联盟（需企业号） | 待验证 | P2 |
| 1688 供应链 | 1688 开放平台（需企业认证） | 待验证 | P2 |

**选品数据流：**
```
用户输入关键词
  → 淘宝联盟 API（商品列表+销量+价格+佣金）
  → 多多进宝 API（商品列表+销量+价格+佣金）
  → 抖音精选联盟 API（商品列表+佣金）
  → 小红书开放平台（笔记量+互动数据）
  → 数据聚合（统一格式+统计+排序）
  → AI 基于以上真实数据做综合分析（竞争格局/进入难度/细分机会）
  → 输出选品报告
```

**不使用蝉妈妈/飞瓜等第三方数据服务**（本质是爬虫+算法估算，非官方授权，商业化有法律风险）。

### 设计原则

1. **选品为根** — 所有模块围绕"选对品、卖好货"展开
2. **用户做选择题** — AI 自动推导参数，用户只在关键节点做审核决策
3. **平台无关架构** — 核心 AI 能力不依赖任何平台 API，平台特定功能通过适配层扩展
4. **AI 越用越准** — 系统学习用户的选品偏好、经营风格，持续优化推荐
5. **模块独立、数据贯通** — 每个模块可独立使用，共享统一数据层
6. **降级容错** — 任何外部 API 失败都有备选方案，永不阻塞用户流程
