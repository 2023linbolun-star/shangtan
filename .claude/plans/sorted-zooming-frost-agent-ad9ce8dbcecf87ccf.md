# 商探AI -- AI人格感与成长体验设计方案

## 架构理解

在给出方案之前，先梳理你的后端已有的进化基础设施：

| 后端能力 | 对应代码 | 前端目前状态 |
|---------|---------|------------|
| User DNA（用户画像：风格/调性/受众/偏好） | `memory.py` - `get_user_dna()`, `update_user_dna()` | `/feedback/dna` 接口存在，但前端未展示 |
| Few-Shot 案例库（用户认可的优秀案例） | `memory.py` - `get_few_shots()`, `add_few_shot()` | 完全不可见 |
| Failure Guardrails（失败教训） | `memory.py` - `get_failure_guardrails()` | 完全不可见 |
| Learning Engine（从反馈中进化） | `learning.py` - `process_feedback()` | 只有简单的"反馈已记录" |
| Agent 人格（不同模块有不同 ROLE prompt） | `listing_agent.py` 的 ROLE, `prompts.py` | 完全不可见 |
| interaction_count / version（DNA版本号） | `UserDNA.interaction_count`, `UserDNA.version` | 完全不可见 |
| confidence（AI 自信度） | `AgentResult.confidence` | 完全不可见 |

核心问题很清晰：**后端有一整套"大脑"，但前端是个"面瘫"。**

## 一、AI Agent 人格感设计

### 1.1 设计原则：专业伙伴，不是聊天机器人

商探AI 是 B 端电商工具，用户是卖家。人格设计的定位：

> **像一个经验丰富的电商老手搭档** -- 不废话、不卖萌，但有温度、有态度、偶尔来一句让你会心一笑的专业判断。

避免的方向：
- 不要可爱卡通形象（B端用户会觉得不靠谱）
- 不要"您好，有什么可以帮您"式的客服语气
- 不要满屏emoji

追求的方向：
- 说人话、有判断力、敢表态
- 像一个坐在你旁边的资深运营，偶尔抬头跟你说一句"这个品类我建议别碰"

### 1.2 各模块人格定义

基于你现有的 Agent 架构（每个 Agent 有独立的 ROLE），前端应该让每个模块有可感知的"专家角色"：

| 模块 | 角色名 | 人格关键词 | 说话风格示例 |
|------|--------|-----------|------------|
| 选品雷达 (scout) | 选品参谋 | 冷静、数据驱动、直言不讳 | "竞争评分 78/100，说实话，这个赛道不好打。但我发现了一个缝隙..." |
| Listing优化 (listing) | SEO老手 | 精确、强迫症、字字计较 | "标题还剩3个字符位，我塞了个长尾词进去，搜索价值+12%" |
| 内容工厂 (content) | 内容操盘手 | 懂流量、有网感、platform-native | "这条抖音脚本我用了'反转钩子'结构，完播率通常更高" |
| 客服AI (cs) | 客服主管 | 耐心、应变快、会化解冲突 | "这类投诉我见过200+次，最佳话术是先共情再给方案" |
| 投放助手 (ads) | 投手老炮 | 精于算账、ROI至上 | "CPC 降到 0.8 了，但转化没跟上，建议收窄人群包" |

### 1.3 人格在界面中的具体体现方式

**方式一：操作结果中的"专家点评"（最核心）**

每次 Agent 返回结果时，除了结构化数据，增加一段自然语言的"专家说"。这不是闲聊，而是解释 AI 为什么这么做。

后端改动：在 `AgentResult.output` 中增加 `expert_comment` 字段。在 `evaluate()` 方法中，让 AI 生成一句 30-50 字的"决策说明"。

前端呈现：
```
┌─────────────────────────────────────────────┐
│  选品参谋 的分析结果                          │
│                                              │
│  [结构化数据表格/图表...]                     │
│                                              │
│  ┌─ 参谋说 ─────────────────────────────┐   │
│  │ 这个品类淘宝端已经红海了，但拼多多     │   │
│  │ 的白牌空间还很大。建议走拼多多+抖音    │   │
│  │ 种草的组合打法。                      │   │
│  └──────────────────────────────────────┘   │
│                                              │
│           [采纳]  [不太对，我来改]            │
└─────────────────────────────────────────────┘
```

**方式二：操作过程中的"思考透传"**

利用你已有的 SSE 流（`/pipeline/{id}/stream`），在 Agent 执行过程中，实时透传 Agent 的 `think()` 阶段推理过程。

```
正在分析...
  > 选品参谋正在扫描 4 个平台的真实数据...
  > 发现 2,347 个相关商品，正在分析价格分布...
  > 识别到你上次说过"不做低价内卷"，已过滤 30 元以下商品
  > 正在生成分析报告...
```

关键：第三行 "识别到你上次说过..." 是人格感的精髓 -- 它让用户意识到 AI 记住了自己的偏好。

**方式三：界面角落的"待命提示"**

不是弹窗，而是在各模块页面顶部有一行低调的状态文案，定期轮换：

- 选品雷达页面："已为你追踪 3 个品类的价格波动，有 1 个值得关注"
- Listing页面："你最近的标题风格偏向属性堆叠，考虑试试场景化标题？"
- 内容页面："你上次生成的小红书笔记被点赞了，我记下了那种风格"

这些提示应由后端新增一个 `/api/agent-tips` 接口生成，基于 User DNA 和最近活动记录。

## 二、AI 成长可视化设计

### 2.1 核心概念："AI 默契度"

不叫"AI等级"、"AI智能度"（太技术化），叫 **"默契度"** -- 暗示这是双向的，你用得越多，我们越默契。

数据来源（全部来自你现有的后端）：
- `UserDNA.interaction_count` -- 交互次数
- `UserDNA.version` -- DNA更新版本
- `FewShotExample` 数量 -- AI 学到了多少"好案例"
- `FailurePattern` 数量 -- AI 从多少错误中学习了
- `AgentMemory` 的 `feedback_score` 统计 -- 好评率趋势

### 2.2 默契度仪表盘

新增 API 端点 `GET /api/feedback/growth-stats`，返回：

```json
{
  "overall_rapport": 72,          // 总默契度 0-100
  "interaction_count": 156,       // 总交互次数
  "dna_version": 23,              // DNA已更新23次
  "learned_preferences": 8,       // 学到了8个偏好
  "good_cases": 12,               // 12个优秀案例入库
  "lessons_learned": 5,           // 5个失败教训
  "approval_rate_trend": [0.6, 0.65, 0.72, 0.78, 0.82],  // 最近5周好评率
  "modules": {
    "scout": {"rapport": 85, "interactions": 67},
    "listing": {"rapport": 65, "interactions": 45},
    "content": {"rapport": 58, "interactions": 34}
  }
}
```

前端呈现：

```
┌──────────────────────────────────────────────┐
│  AI 默契度                                    │
│                                               │
│  ████████████████████░░░░░░  72/100           │
│                                               │
│  已交互 156 次 | DNA v23 | 学会 8 个偏好       │
│                                               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│  │ 选品参谋  │ │ SEO老手  │ │ 内容操盘手│      │
│  │  85/100  │ │  65/100  │ │  58/100  │      │
│  │ ████████ │ │ ██████   │ │ █████    │      │
│  └──────────┘ └──────────┘ └──────────┘      │
│                                               │
│  好评率趋势                                    │
│  82% ──────────╮                              │
│  60% ──╯        现在                          │
│                                               │
│  AI 最近学会了:                                 │
│  - "不要在标题中使用'最'字" (3天前)            │
│  - "偏好小红书种草风格" (1周前)                │
│  - "目标客群: 25-35岁女性" (2周前)             │
└──────────────────────────────────────────────┘
```

### 2.3 成长里程碑

在默契度达到特定阈值时，触发一次性的里程碑通知（非弹窗，而是在页面顶部一个可折叠的条幅）：

| 默契度 | 里程碑名称 | 通知文案 |
|--------|----------|---------|
| 10 | 初次见面 | "初步了解了你的店铺风格，接下来会越来越准" |
| 30 | 开始默契 | "已经掌握了你的 3 个核心偏好，生成质量正在提升" |
| 50 | 得力助手 | "学习了 6 个你认可的优秀案例，现在能模仿你的风格了" |
| 70 | 心有灵犀 | "已从 4 次失误中学到教训，同类错误不会再犯" |
| 90 | 默契搭档 | "156 次协作，好评率 82%。我们是很好的搭档" |

### 2.4 每次反馈后的"学习回执"

当前用户点击"反馈已记录"后，返回的是一个无感知的确认。应改为：

**正面反馈后：**
```
已记录。这类风格我记住了，下次会优先参考。
[查看 AI 学到了什么]  // 点击展开，显示新增的 few-shot 案例
```

**负面反馈后（附带修改意见时）：**
```
收到。我从你的修改中学到了："不要在小红书笔记中使用过多营销语气"。
以后会注意。
[查看 AI 的教训清单]  // 点击展开，显示 failure_guardrails 列表
```

**大幅编辑后（edit_distance > 0.3）：**
```
你改了不少内容。我正在分析你的修改，学习你的风格偏好...
[学习完成] 用户画像已更新：偏好语气 -> 自然口语化
```

## 三、8 个具体的 Delight Moments

### Moment 1: "我记得你说过"

**触发时机：** Agent 在 `observe()` 阶段加载 User DNA 或 failure_guardrails 时，将"因为你的偏好而做了什么调整"写入结果的 `reasoning` 字段。

**前端表现：** 在结果卡片上方，用淡色背景条显示：
```
已根据你的偏好调整：过滤了低于 30 元的商品 | 避免了"最"字的使用 | 采用了你偏好的种草风格
```

**为什么有效：** 用户会想"它居然记得我说过不做低价品"，这是最强的"懂我"信号。

**后端改动：** 在 `build_agent_system_prompt` 的输出中增加一个 `applied_preferences` 列表，记录本次实际应用了哪些 DNA/guardrail。

### Moment 2: "比上次更好了"

**触发时机：** 同类任务（相同 agent_type + 相似 keyword）的 confidence 分数比上次高时。

**前端表现：** 结果卡片右上角一个小徽章：
```
置信度 85% (+12%) -- 这次比上次更确定了
```

**为什么有效：** 用户看到具体的进步数字，感受到 AI 在成长。

**后端改动：** 在 `_record_execution` 中，查询该用户同类型上一次执行的 confidence，计算差值返回。

### Moment 3: "主动预警"

**触发时机：** 当用户输入的关键词/品类触发了已有的 failure_guardrail 时。

**前端表现：** 在提交前（或提交后立即），弹出一个非阻塞的提示条：
```
提醒：上次做"防晒霜"相关内容时，你不满意营销感太重。
这次我会特别注意控制语气。  [好的] [这次不用管]
```

**为什么有效：** AI 主动提起过去的教训并承诺改进，像一个真正吸取经验的搭档。

### Moment 4: "第一次懂你"

**触发时机：** User DNA 从空白到首次有实质性内容时（`version` 从 0 到 1-3 之间的某次更新）。

**前端表现：** 在默契度区域显示一个一次性的"成长通知"：
```
我开始了解你了 --
你的目标客群是 25-35 岁女性，品牌调性偏向自然简约。
如果不对，你随时可以修正。  [很准] [帮我改改]
```

**为什么有效：** 用户第一次看到 AI "说出"对自己的理解，会产生强烈的"被看见"感。

### Moment 5: "跨模块联动"

**触发时机：** 用户在选品模块发现了一个好品，切换到内容模块时。

**前端表现：** 内容模块自动提示：
```
内容操盘手注意到你刚在选品雷达中关注了"夏季防晒伞"。
需要我为这个品生成种草内容吗？ [好的] [先不用]
```

**为什么有效：** 跨模块记忆打破了"每个模块是独立工具"的感觉，让用户觉得背后是一个完整的"大脑"。

**后端改动：** 利用已有的 `event_bus.py`，在 scout 完成时发事件，content 模块订阅并生成推荐。

### Moment 6: "成长回顾"

**触发时机：** 用户每使用系统满 50 次交互，或每两周登录时。

**前端表现：** 在 Dashboard 顶部显示一个可展开的回顾卡片：
```
过去两周，AI 进步了:
- 好评率从 65% 提升到 78%
- 学会了 2 个新偏好："避免过度使用emoji" "标题优先放品类词"
- 新增了 3 个优秀案例入库
- 默契度: 58 -> 67 (+9)

你们正在变成更好的搭档。
```

### Moment 7: "风格指纹"

**触发时机：** 当 User DNA 积累了足够丰富的数据后（version >= 10）。

**前端表现：** 在设置页或默契度仪表盘中，展示一个"AI眼中的你"画像：
```
┌─ AI 眼中的你 ──────────────────────────────┐
│                                              │
│  店铺调性:  自然简约 ■■■■■■■■□□              │
│  价格策略:  中高端   ■■■■■■■□□□              │
│  内容偏好:  种草 > 评测 > 开箱                │
│  风险偏好:  稳健派（避开红海品类）             │
│  平台重心:  小红书 40% / 抖音 35% / 淘宝 25%  │
│                                              │
│  独特标签: #不做低价内卷 #偏爱场景化标题       │
│                                              │
│  这是 AI 通过 156 次协作学习到的画像。         │
│  觉得不准？ [帮我修正]                        │
└──────────────────────────────────────────────┘
```

**为什么有效：** 用户看到 AI 对自己的"理解"被可视化出来，既有趣又实用。而且"觉得不准？帮我修正"把控制权交给用户，不会让人不舒服。

### Moment 8: "老手的自信与坦诚"

**触发时机：** Agent 的 `evaluate()` 返回了较低的 confidence（< 0.5）时。

**前端表现：** 在结果展示中，不是隐藏低信心，而是坦诚说明：
```
SEO老手提醒: 这次结果我不太确定（置信度 45%）。
原因是这个品类我接触不多，优秀案例库里还没有相关参考。
建议你仔细审核，你的修改会帮助我学习。
```

**为什么有效：** 反直觉地，AI 主动承认不确定，比假装什么都行更让人信任。它传达的是"我是诚实的"，而不是"我什么都会"。同时引导用户反馈，形成学习闭环。

## 四、让 AI 从"工具"变成"伙伴"的设计

### 4.1 伙伴的核心特征：主动性

工具等你操作，伙伴会主动提议。关键是"主动但不烦"。

**实现策略：分层通知机制**

| 层级 | 方式 | 内容示例 | 频率限制 |
|------|------|---------|---------|
| 静默层 | 页面角落的小数字/圆点 | "2条新洞察" | 无限制 |
| 低调层 | 页面顶部可折叠横幅 | "你关注的品类有价格波动" | 每天最多3条 |
| 主动层 | 模块入口处的推荐卡片 | "建议为'防晒伞'生成内容" | 每天最多1条 |
| 紧急层 | 显眼但非弹窗的警告 | "你的淘宝标题触发了新规违规" | 仅必要时 |

永远不用弹窗。永远不在操作流程中打断用户。

### 4.2 后端支撑：新增 "Proactive Insights" 接口

```
GET /api/agent-insights
```

后端定时任务（或用户登录时触发）分析：
- 用户关注品类的价格/销量变化
- 新的平台规则变化对用户现有标题的影响
- 跨模块联动机会（选品 -> 内容 -> 投放）
- 基于 DNA 的个性化建议

返回格式：
```json
{
  "insights": [
    {
      "type": "market_change",
      "module": "scout",
      "priority": "low",
      "title": "你关注的'防晒伞'品类出现新机会",
      "detail": "拼多多端竞品减少 15%，可能是进入窗口期",
      "action": {"label": "查看分析", "route": "/scout?keyword=防晒伞"},
      "created_at": "2026-03-25T10:00:00Z"
    },
    {
      "type": "cross_module",
      "module": "content",
      "priority": "medium",
      "title": "建议为'夏季凉席'生成小红书内容",
      "detail": "这个品你3天前选好了，小红书种草热度正在上升",
      "action": {"label": "一键生成", "route": "/content?product_id=xxx"},
      "created_at": "2026-03-25T09:00:00Z"
    }
  ]
}
```

### 4.3 "伙伴感"的文案设计原则

**对比：工具语气 vs 伙伴语气**

| 场景 | 工具语气 | 伙伴语气 |
|------|---------|---------|
| 执行完成 | "分析完成" | "分析好了，这个品类比我想的竞争更激烈" |
| 等待中 | "加载中..." | "正在扫描 4 个平台的数据，大约需要 15 秒" |
| 报错 | "请求失败，请重试" | "这个平台的接口暂时不通，我先用其他平台的数据给你分析" |
| 空结果 | "暂无数据" | "这个关键词在各平台都没找到数据，换个近义词试试？我建议用'XX'" |
| 反馈确认 | "反馈已记录" | "记住了。下次做同类内容时我会注意这一点" |

### 4.4 让用户感觉"AI 在为我工作"而不是"我在使用 AI"

**Pipeline 进度条的人格化改造：**

当前 Pipeline SSE 推送的是纯状态数据。改为同时推送 Agent 的"自述"：

```
┌── 全链路分析进行中 ───────────────────────────┐
│                                                │
│  [====>        ] 第2步/共4步                    │
│                                                │
│  Step 1  选品雷达   完成                        │
│  选品参谋: 扫描了 2,347 个商品，找到 3 个蓝海    │
│  细分方向。报告已就绪。                          │
│                                                │
│  Step 2  Listing优化  进行中...                  │
│  SEO老手: 正在为抖音和淘宝分别优化标题。         │
│  你之前说过不喜欢堆砌式标题，这次用场景化写法。   │
│                                                │
│  Step 3  内容生成   待执行                       │
│  Step 4  发布排期   待执行                       │
└────────────────────────────────────────────────┘
```

## 五、实施路径（按优先级排序）

### Phase 1 -- 快速见效（1-2周）

**后端改动：**

1. **新增 `GET /api/feedback/growth-stats` 接口** -- 聚合 UserDNA、FewShot、FailurePattern、AgentMemory 数据，计算默契度分数

## 七、文案风格指南

### 总原则
1. **说人话** -- 不说"系统检测到"，说"我发现了"
2. **有判断** -- 不说"以下是结果"，说"结果出来了，这个品类比我想的更值得做"
3. **有记忆** -- 尽可能引用过去的交互（"你上次说过..."、"你之前偏好..."）
4. **坦诚** -- 不确定的时候说不确定，比假装什么都行更赢得信任
5. **不废话** -- B 端用户时间宝贵，每句话都要有信息量

### 禁止用语
- "亲"、"亲爱的用户"
- "有什么可以帮您？"
- 过多的emoji
- "请您耐心等待"

### 推荐用语风格
- "发现了..."、"建议..."、"提醒一下..."
- "我记得你说过..."、"基于你的偏好..."
- "说实话，这个..."、"坦率地说..."
- "下次我会注意..."、"从上次的教训中..."
